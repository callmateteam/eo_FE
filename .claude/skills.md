# 레시피북

항상 가장 작은 레시피를 선택한다.

## 기능 추가

1. 대상 라우트와 공용 컴포넌트를 먼저 읽는다.
2. 기존 레이아웃·스타일 단위를 최대한 재사용한다.
3. 필요한 상태와 props만 추가한다.
4. 여러 파일 수정 시 `lint` + 타입 확인, 구조 변경 시 빌드 확인.

## 버그 수정

1. 코드에서 실패 경로를 추적한다.
2. 잘못된 가정이나 상태 전환이 발생하는 지점을 찾는다.
3. 가장 작은 안전한 수정으로 고친다.
4. 공용 코드·타입 경계를 건드렸다면 `lint` + 타입 확인.

## 스타일링

1. 주변 코드의 간격·타이포그래피·색상 패턴을 먼저 확인한다.
2. 기존 Tailwind 클래스와 토큰을 우선 사용한다.
3. 모바일·데스크톱에서 일관성 유지.
4. 명시적 요청 없이 새로운 시각 언어를 도입하지 않는다.

## 리팩터링

1. 의미 있는 중복이 두 곳 이상인지 확인한다.
2. 추출 후 더 명확해지는 부분만 추상화한다.
3. 런타임 동작과 외부 인터페이스는 유지한다.
4. 타입 구조 변경 시 타입 오류 확인.

## 리뷰

1. 변경 파일을 먼저 읽는다.
2. 심각도와 영향이 드러나는 구체적 발견 사항을 적는다.
3. 테스트 누락·깨지기 쉬운 가정·동작 드리프트를 지적한다.
4. 요약은 발견 사항보다 앞세우지 않는다.
