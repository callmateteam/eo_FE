# Skills

이 파일은 이 저장소에서 반복되는 Codex 작업을 위한 "레시피북"입니다.
항상 작업에 맞는 가장 작은 레시피를 고릅니다.

## 기능 추가 레시피

새로운 섹션, 컴포넌트, 페이지 동작을 추가할 때 사용합니다.

1. 대상 라우트와 인접한 공용 컴포넌트를 먼저 확인합니다.
2. 기존 레이아웃과 스타일 기본 단위를 최대한 재사용합니다.
3. 요청된 동작에 필요한 상태와 props만 추가합니다.
4. 여러 파일을 건드렸다면 `npm run lint` 또는 제한된 빌드 검증을 실행합니다.

## 버그 수정 레시피

이미 동작이 잘못된 상황에서 사용합니다.

1. 코드에서 실패 경로를 추적합니다.
2. 가장 먼저 잘못된 가정이나 상태 전환이 발생하는 지점을 찾습니다.
3. 그 지점을 가장 작은 안전한 수정으로 고칩니다.
4. 영향 경로를 검증하고, 공용 코드를 건드렸다면 `lint`도 실행합니다.

## 스타일링 레시피

핵심 동작은 유지한 채 시각 요소를 조정할 때 사용합니다.

1. 주변 코드의 간격, 타이포그래피, 색상 사용 방식을 먼저 확인합니다.
2. 기존 토큰, 유틸리티, 클래스 패턴을 우선 사용합니다.
3. 모바일과 데스크톱에서 시각 변경이 일관되게 보이도록 유지합니다.
4. 명시적 요청이 없다면 새로운 시각 언어를 도입하지 않습니다.

## 리팩터링 레시피

중복을 줄이거나 가독성을 높일 때 사용합니다.

1. 의미 있는 중복이 최소 두 군데 이상 있다는 것을 확인합니다.
2. 추출 후 더 명확해지는 부분만 추상화합니다.
3. 요청이 없는 한 런타임 동작과 외부 인터페이스는 유지합니다.
4. 기회주의적 리팩터링 대신 영향 구간만 검증합니다.

## 리뷰 레시피

사용자가 리뷰나 감사를 요청했을 때 사용합니다.

1. 변경 파일부터 먼저 읽습니다.
2. 심각도와 영향이 드러나는 구체적 발견 사항을 적습니다.
3. 테스트 누락, 깨지기 쉬운 가정, 동작 드리프트를 지적합니다.
4. 요약은 짧게 두고, 발견 사항보다 앞세우지 않습니다.
